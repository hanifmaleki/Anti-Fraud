Description

A rule-based system ....

  1. Giving into account transactions history
  2. Introduce trx types
  3. Distinguish intelligently amongs different types of transactions

List of transactions properties

  1. Transaction date
  2. Transaction amount //TODO BigDecimal
  3. Transaction applications (Insurance, Food, Transfer, Online-Shop-real-ware, Online-Shop digital-ware,....)
  4. IP address
  5. Country

The philosophy behind the rules is that the probability of fraud in different types of the transactions are not the same e.g. For type insurance the probability is the lowest while a frauder is likely to try some transaction of type transfer

The expected amount is also different in each type of transaction. While buying real ware of price 180 seems norma, the same amount need manual verification on a transaction of type food.

Objectives

Each transaction is queried considering last 3 transaction of a given credit card.

Enrich the fraud detection service considering the following rules:

  1. If the transactions in the last 24 hours are from more than 2 countries => Fraud
  2. If the transactions in the last 24 hours are from more than 3 IPs => Fraud
  3. If the transactions in the last 24 hours are from 2 countries => Personal
  4. If the transactions in the last 24 hours are from 3 IPs => Personal
  5. If a transaction is known as manual, the next transactions would be either manual or prohibited (unless ....)
  6. If a transaction is known as fraud, the next transactions would be fraud (unless ....)
  7. Rules on transactions amounts for each transactions based on the type
    Transaction Type Most Allowed Most Manual
    insurance 400 1200
    real ware 200 500
    food 150 300
    online ware 150 300
    transfer 300 900
