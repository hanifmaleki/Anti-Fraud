Description

An enterprise anti-fraud system has hundreds of users who take advantage of the system by only checking validity of the transactions. Such kind of users do not need to modify and see list of stolen cards, suspicious IP addresses and even other users. On the other hand there are very limited number of users eho are completely trusted by users and are allowed to access and modify more sensitive data.

In this stage authentication and authorization ...

  1. for this reason, it is necessary to define password for the users ...
  2. Why not maintaining passwords as simple text
  3. What is hash function ... MD5 (for simplicity)

Objectives

  1. Enrich the User model by adding field password ...
  2. Passwords are immediately encoded ...
  3. Add an endpoint to user controller for changing password ...
  4. Add a preconfigured admin user admin with password admin should be added by default.
  5. Normal user can only do the following:
    • Change her/his password
    • Ask for the validity of a transaction
    • See the list of stolen cards
    • Add a stolen card
    • See the list of suspicious IPs
    • Add suspicious IPs
  6. Updating/Adding/Removing other users, removing suspicious IP and stolen cards are only done by the admin users.
  7. Enrich fraud detection service to prohibit stolen cards and suspicious IPs
